React app workflow diagram showing how files connect from App.tsx to the API

Entry / Page
Component
Hook
Service / API
1
App.tsx
App entry point
The root of your app. React starts here. It renders CollegeRanksPage — think of it as the shell that holds all pages.
↓
2
pages/CollegeRanksPage.tsx
Page — layout only
Owns the page title and layout. Has no logic of its own — just renders CollegeSearch inside it. Pages compose components, they don't do work themselves.
↓
3
components/search/CollegeSearch.tsx
Component — user interaction
Owns the text input and Search button. When user types and clicks Search, it stores the submittedId in state and passes it down to CollegeRankGrid.
↓
4
components/grid/CollegeRankGrid.tsx
Component — display only
Receives collegeId as a prop. Calls the hook to get data, then renders the MUI DataGrid. This component only cares about displaying — not fetching.
↓
5
hooks/useCollegeRanks.ts
Hook — logic & state
Manages rows, loading, and error state. Calls the service when collegeId changes. Returns data back to the grid component.
↓
6
services/collegeService.ts
Service — API call
Makes the actual axios.get call to your backend at ai-college.local:8080. Returns raw data. Has no knowledge of React — pure data fetching only.
↓
7
models/college.ts
Model — data shape
Defines what a CollegeRank object looks like using TypeScript. Used across hook, service, and grid so everyone agrees on the data structure.
